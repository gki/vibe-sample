# 共通PRD

このドキュメントには、プロジェクト全体で共通する項目を定義します。

## 対象ユーザーとペルソナ

### プライマリユーザー層

インターネットの利用に慣れている学生・社会人ユーザーを想定しています。

#### ユーザー特性

- **技術習熟度**: インターネットの基本的な操作ができる
- **デバイス**: スマートフォンおよびブラウザ（PC）の両方で利用する
- **データ管理**: このプロジェクトはサンプルプロジェクトのため、ユーザー別のデータ管理は行わない（全ユーザーでデータを共有）

#### セカンダリユーザー層

セカンダリユーザー層は定義しません。

## 用語集

プロジェクト全体で使用される専門用語の定義を記載します。

- **task**: todoで行うタスクを指す
- **title**: taskの内容
- **complete**: 完了したことを表現する

## 共通UI/UXルール

プロジェクト全体で統一すべきUI/UXのルールを定義します。

### 通信中、通信エラー時の共通ルール

#### 画面表示時の通信（データ取得時）

React Suspenseを利用し、ローディング中のスケルトンを表示します。

- データ取得中はSuspenseのfallbackとしてスケルトンUIを表示
- スケルトンUIは、実際のコンテンツの構造に近い形状で表示する

#### 画面操作時の通信（データ送信時）

操作したUI要素をdisableにし、ローディングインジケータを表示して処理中であることを示します。

- ボタンやフォームなどの操作UI要素を`disabled`属性で無効化
- 処理中であることを示すローディングインジケータを表示（例：ボタン内にスピナーアイコン）
- 複数の操作UIがある場合、操作した対象のUI要素のみをdisableにする

#### 通信エラー時

ブラウザの`alert`でユーザーにフィードバックします。

- 通信エラーが発生した場合、`window.alert()`を使用してエラーメッセージを表示
- エラーメッセージは簡潔で分かりやすい内容とする

### 共通フィードバックUIルール

#### エラーフィードバック

validationエラーおよび通信エラーは、ブラウザの`alert`を使用してユーザーにフィードバックします。

- **validationエラー**: 入力値の検証エラーが発生した場合、`window.alert()`でエラーメッセージを表示
- **通信エラー**: API通信エラーが発生した場合、`window.alert()`でエラーメッセージを表示

#### ダイアログ・モーダル

独自のダイアログやモーダルコンポーネントは実装しません。ブラウザ標準機能のみを使用します。

- ブラウザ標準の`window.alert()`、`window.confirm()`、`window.prompt()`のみを使用
- カスタムのモーダルUIコンポーネントは実装しない

#### トースト通知

独自のトースト通知コンポーネントは実装しません。

- カスタムのトーストUIコンポーネントは実装しない
- 通知が必要な場合は、ブラウザ標準の`alert`を使用する
